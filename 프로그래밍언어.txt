## 언어의 실행에 따른 종류

한번에 실행되는지 한줄씩 실행되는지에 따라서 나뉩니다 

1. 컴파일 언어
    - C, C++, JAVA ....
    - 숫자만 입력받을 수 있습니다.
        - 0 or 1 - 다른말로 이진수(바이너리)라고 부릅니다.
        - 0과 1로 이루어진 값들을 (기계어)
    - 기계어로 어딘가에 바뀌어서 저장이 되어 있다가
    - 내가 실행하면 한번에 실행이 됩니다 char string int float
    - 무겁겠죠. 엄격하게 만들어져야겠죠.
    - 실행은 이미 다 바뀌어 있는 상태에서 작동하기 때문에 속도가 빠릅니다.
    - 문자로 작성된 소스코드(파일)을 기계어로 바꾸는 작업을 한꺼번에 하는 과정 → 컴파일(번역)
      기계어 어셈블리어

2. 인터프리터 언어(스크립트 언어)
    - 자바스크립트, 파이썬, 루비, PHP....
    - 한줄한줄 해석하기 때문에 컴파일언어보다 느린 편입니다
    - 가볍다. 조금 헐렁하게 만들어져도 되겠죠. var number1
    - 컴파일 과정
    - 없이 바로 실행이 가능합니다.
    - 이 과정을 인터프리터가 처리를 해줍니다.
      추상화 수준이 높은 것. 여러가지가 많다. 
        

REPL (Read, Evaluation, Print, Loop)

- 결국 인터프리터는 REPL 과정을 따라서 동작을 하게 됩니다.
- 컴파일(번역)과정이 없는것이 아니라 눈에 보이지 않을 뿐입니다.
- Read: 입력된 명령어를 읽고
    - 이 과정에서 인터프리터가 번역을 합니다 => 기계어
    - 문자로 된 명령어를 => 숫자(기계어/binary)로 변환
- Evaluation: 명령어를 실행
- Print: 명령어를 실행한 결과를 **돌려줍니다**
- Loop: 이 과정을 계속 반복할 수 있습니다.
